Introduction to Log Analysis — Foundation of SOC
Dec 10, 2025 • Category: Logs • Reading: 6 min

Why logs matter
Logs are the primary evidence for security investigations. They capture what happened, where and who performed actions. Good log coverage + reliable timestamps = faster triage and more accurate detections.

Common log sources

  • Windows: Event Logs — Security, System, Application
  • Linux: syslog, auth.log, journalctl
  • Network: Firewalls, routers — connection and drop logs
  • Cloud: CloudTrail, CloudWatch, Azure Activity Logs

Quick commands

sudo journalctl -u ssh -p err --since "1 hour ago"
# grep failed logins
sudo grep -i "failed password" /var/log/auth.log | tail -n 50

# Windows PowerShell: recent security events
Get-WinEvent -FilterHashtable @{LogName='Security'; StartTime=(Get-Date).AddHours(-6)} |
  Select TimeCreated, Id, LevelDisplayName | Format-Table -AutoSize

Beginner tips

  1. Centralize logs to a SIEM or ELK stack.
  2. Normalize fields so queries work across sources.
  3. Ensure clocks are synchronized (NTP).
  4. Tune alerts to avoid noise and false positives.
